# 正午の海面気圧による日次標高計画

## 目的

東京の当日12時の海面気圧と、取得時点の BME280 の気温・測定気圧から標高を1日1回算出・保存し、気圧変動による日内ドリフトを防ぐ。

## フェーズ

### Phase 1: 仕様更新

- 気象庁のAMeDAS JSON、取得時刻、保存値、失敗時の動作を定義する。
- 旧方式の ADR を退避し、新方式の ADR を作成する。

状態: 完了

### Phase 2: 取得・計算・保存

- 13:00 JST 以降に東京の当日12時の海面気圧を取得する。
- 取得時点の BME280 値で標高を算出する。
- 海面気圧、対象日、算出標高を EEPROM に保存する。
- 現行 EEPROM 形式から設定値を維持して移行する。

状態: 完了

### Phase 3: 表示・送信

- OLED と `home.weather.altitude` で保存済み標高を使用する。
- センサー更新ごとの標高再計算を廃止する。

状態: 完了

### Phase 4: 検証

- `python3 tests/test_altitude_and_jma.py` を実行する。
- `git diff --check` を実行する。
- `arduino-cli compile --fqbn esp32:esp32:nano_nora .` を実行する。
- 実機で更新日時、OLED、New Relic を確認する。

状態: 自己検証、`git diff --check`、Nano ESP32 向けコンパイルは完了。実機確認は未実施。

## 成功基準

- 東京の当日12時の海面気圧を13:00 JST以降に取得する。
- 取得成功時だけ標高を算出・保存し、翌日の更新まで固定利用する。
- 失敗時は前回値を維持し、1時間間隔で再試行する。
- OLED と New Relic が同じ保存済み標高を使用する。
- Nano ESP32 向けコンパイルが成功する。
- 実機で表示と送信を確認できる。
