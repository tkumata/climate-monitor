# ADR-006: 正午の海面気圧から日次標高を保存する

## 状態

採用

## 背景

前日の海面平均気圧と現在の BME280 測定値を使って毎秒標高を再計算すると、気象による気圧変化が標高変化として現れ、実際には移動していない端末でも負の値を含む日内変動が発生する。

当日詳細版 `data1s.html` では正午の海面気圧を取得できず、過去の気象データ検索は当日分を公開しないため、東京観測所の AMeDAS JSON を使用する。

## 決定

13:00 JST 以降に東京観測所番号 `44132` の当日12:00の `normalPressure` を取得する。取得時点の BME280 の気温と測定気圧から標高を1回算出し、海面気圧、対象日、標高を EEPROM に保存する。

OLED と New Relic は保存済み標高を翌日の更新まで使用し、センサー更新ごとの再計算は行わない。取得または計算に失敗した場合は前回値を維持し、1時間後に再試行する。

## 結果

- 気圧変化による標高の日内ドリフトを防げる。
- 取得失敗や再起動後も最後に成功した標高を使用できる。
- 東京と設置場所の気圧差、および12時観測と取得時点の時間差は推定誤差として残る。
- 公開 HTML の構造変更時には解析処理の更新が必要になる。
