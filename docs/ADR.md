# ADR-010: 最新 Grafana ペイロードを Arc Gauge で表示する

## 状態

採用

## 背景

Grafana Cloud へ5分間隔で送信する最新 OTLP JSON を端末でも確認し、人が読みやすい気象表示として利用したい。履歴保存は不要で、最新1件だけが必要である。

## 決定

Grafana 送信用の既存4KiB固定長バッファを最新ペイロードとして RAM に保持し、生成のたびに同じバッファを上書きする。EEPROM とファイルシステムには保存しない。

Digest 認証付きの `/api/grafana/latest` で最新 JSON を返し、`/weather` はブラウザ標準の JavaScript、CSS、SVG だけで8個の Arc Gauge を描画する。WBGT は既存の25°C、28°C、31°C基準で色を変える。

## 結果

- 追加のペイロード複製、履歴管理、外部チャートライブラリを必要としない。
- 最新の送信試行内容をラベルと単位付きで確認できる。
- 再起動すると最新ペイロードは消去され、次の有効な送信試行まで待機表示となる。
- 表示範囲を変更する場合はファームウェア更新が必要になる。
