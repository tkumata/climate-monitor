# ADR-005: 気象庁の前日海面平均気圧から標高を算出する

## 状態

置換済み

## 背景

BME280 の高度 API は固定の基準海面気圧から高度を推定している。実際の海面気圧は日々変化するため、測定気温、測定気圧、東京の海面気圧を使って標高を算出し、OLED と New Relic に反映する必要がある。

指定された当日詳細版 `data1s.html` では、日中は海面平均が未確定で空欄となる。前日分は翌日13時40分頃に確定するため、同じ気象庁データ系列の前日詳細版 `data2s.html` を14時以降に取得する。

## 決定

東京の前日海面平均気圧を1日1回 HTTPS で取得し、対象日とともに EEPROM へ保存する。HTML は固定長バッファでストリーム解析し、対象日、地点、列、品質記号、値の範囲を検証する。

標高は BME280 の測定気温と測定気圧、保存済み海面気圧から気温を含む気圧高度式で算出する。算出値を OLED と既存の `home.weather.altitude` へ使用し、固定基準気圧による `bme.readAltitude()` は廃止する。

## 結果

- 日々の海面気圧を反映した推定標高を表示・送信できる。
- 電源断や一時的な取得失敗でも最後に取得した値を利用できる。
- 前日の平均海面気圧と現在の室内測定値を混在させるため、標高が気圧変化に追従して日内変動する。
