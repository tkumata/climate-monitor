# climate-monitor

BME280 は温度・相対湿度・気圧が測れるセンサーです。SSD1306 は 128x64 ピクセルの有機 EL ディスプレイです。これらを用いて単純な温湿度計にするだけではもったいないので、生活に役立つデータに変換することで、マイコンとセンサーを活用できればと思います。生活に役立つデータとしては以下を考えました。

1. 絶対湿度: 部屋干し洗濯物の乾きやすさの目安に活用
2. VPD: 室内植物の蒸散のしやすさで、水やりの目安に活用
3. 不快指数: エアコン稼働の目安に活用
4. 日付・曜日・時間

## 仕様

`loop()` は 100ms 間隔で回し、以下を都度表示します。

### 日付・曜日・時間 

現在市販されているマイコンボードには RTC を保持するための CMOS バッテリーはありません。それゆえ電源の ON/OFF で初期化されてしまいます。また、最近のマイコンボードには Wi-Fi チップが搭載されているので、これを用いて電源 ON 時に NICT や NTT の NTP サーバーから現在時刻を取得し、マイコンの RTC に日本標準時としてセットします。

### 絶対湿度と部屋干しの目安

絶対湿度とは、空気中に含まれる水蒸気の量を表す指標で、通常は「g/m³」（空気 1 立方メートルあたりの水蒸気量）で表されます。絶対湿度が高いほど空気中の水分が多く、乾燥しにくい環境といえます。絶対湿度が低いと単位体積あたりの水分量が少ないため、洗濯物の表面から水分が蒸発しやすくなります。逆に絶対湿度が高いと乾きづらくなります。

絶対湿度と乾きやすさの目安は以下のとおりです。

- 絶対湿度 10 g/m³ 以下: 乾燥しやすい（特に冬場や乾燥した地域）。
- 絶対湿度 10～15 g/m³: 中程度の乾燥速度（気温や風次第）。
- 絶対湿度 15 g/m³ 以上: 乾燥しにくい（特に梅雨や熱帯地域）。エアコンなどで相対湿度を下げるなど対処が必要。

### VPD と水やりの目安

VPD は、空気中の実際の蒸気圧と飽和蒸気圧の差を表し、単位は kPa（キロパスカル）で示します。簡単に言うと、空気がどれだけ乾燥しているかを示し、植物がどれだけ水を蒸散しやすいかを判断する指標です。

高い VPD: 空気が乾燥しており、植物が水を多く蒸散する（給水が必要な場合が多い）。
低い VPD: 空気が湿っており、蒸散が抑えられる（水やりが不要な場合が多い）。

植物の種類や成長段階（発芽、成長期、開花期など）によって最適な VPD は異なりますが、一般的な目安は以下のとおりです。

- 一般的な植物（野菜、花、観葉植物）
  - 発芽期：0.4～0.8 kPa（湿った環境が好ましい）
  - 成長期：0.8～1.2 kPa（蒸散と成長のバランスが良い）
  - 開花・結実期：1.0～1.5 kPa（やや乾燥気味で栄養吸収を促進）
- 熱帯植物（例：ラン）：0.5～1.0 kPa（高湿度を好む）
- 乾燥に強い植物（例：サボテン）：1.5～2.0 kPa でも耐える

### 不快指数とエアコンの目安

不快指数（DI: Discomfort Index）は、気温と相対湿度を基に算出される指標で、人間が感じる暑さや不快感を数値化したものです。

- DI < 60: 寒いと感じる。エアコン（暖房）が必要な場合も。
- DI 60～65: 涼しく快適。エアコン不要。
- DI 65～70: 快適な範囲。ほとんどの人がエアコン不要。
- DI 70～75: やや暑く感じる。一部の人には不快。扇風機や通気で対応可能。
- DI 75～80: 暑さを感じる。エアコン（冷房）を検討。特に高齢者や暑さに弱い人は入れるべき。
- DI 80～85: かなり暑い。ほとんどの人がエアコンを入れた方が快適。
- DI 85 以上: 非常に不快で熱中症リスクが高まる。エアコンを強く推奨。

- Arduino Nano ESP32

### 配線

| SSD1306 | Board |
| ------- | ----- |
| VDD     | 3V3   |
| GND     | GND   |
| SCK     | A5    |
| SDA     | A4    |

| BME280   | Board |
| -------- | ----- |
| VCC      | 3V3   |
| GND      | GND   |
| CS       | D10   |
| MISO     | D12   |
| SCL/SCK  | D13   |
| SDA/MOSI | D11   |
