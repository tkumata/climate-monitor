# climate-monitor

BME280 は温度・相対湿度・気圧が測れるセンサーです。SSD1306 は 128x64 ピクセルの有機 EL ディスプレイです。これらを用いて単純な温湿度計にするだけではもったいないので、生活に役立つデータに変換することで、マイコンとセンサーを活用できればと思います。生活に役立つデータとしては以下を考えました。

1. 絶対湿度: 部屋干し洗濯物の乾きやすさの目安に活用
2. VPD: 室内植物の蒸散のしやすさで、水やりの目安に活用
3. 不快指数: エアコン稼働の目安に活用
4. WBGT: 熱中症予防の目安に活用
5. 日付・曜日・時間
6. 東京の海面気圧で補正した推定標高

## 仕様

`loop()` は 100ms 間隔で回し、以下を都度表示します。

### 日付・曜日・時間

現在市販されているマイコンボードには RTC を保持するための CMOS バッテリーはありません。それゆえ電源の ON/OFF で初期化されてしまいます。また、最近のマイコンボードには Wi-Fi チップが搭載されているので、これを用いて電源 ON 時に NICT や NTT の NTP サーバーから現在時刻を取得し、マイコンの RTC に日本標準時としてセットします。

### 絶対湿度と部屋干しの目安

絶対湿度とは、空気中に含まれる水蒸気の量を表す指標で、通常は「g/m³」(空気 1 立方メートルあたりの水蒸気量) で表されます。絶対湿度が高いほど空気中の水分が多く、乾燥しにくい環境といえます。絶対湿度が低いと単位体積あたりの水分量が少ないため、洗濯物の表面から水分が蒸発しやすくなります。逆に絶対湿度が高いと乾きづらくなります。

絶対湿度と乾きやすさの目安は以下のとおりです。

- 絶対湿度 10 g/m³ 以下: 乾燥しやすい (特に冬場や乾燥した地域)。
- 絶対湿度 10～15 g/m³: 中程度の乾燥速度 (気温や風次第)。
- 絶対湿度 15 g/m³ 以上: 乾燥しにくい (特に梅雨や熱帯地域)。エアコンなどで相対湿度を下げるなど対処が必要。

### VPD と水やりの目安

VPD は、空気中の実際の蒸気圧と飽和蒸気圧の差を表し、単位は kPa (キロパスカル) で示します。簡単に言うと、空気がどれだけ乾燥しているかを示し、植物がどれだけ水を蒸散しやすいかを判断する指標です。

高い VPD: 空気が乾燥しており、植物が水を多く蒸散する (給水が必要な場合が多い)。
低い VPD: 空気が湿っており、蒸散が抑えられる (水やりが不要な場合が多い)。

植物の種類や成長段階（発芽、成長期、開花期など）によって最適な VPD は異なりますが、一般的な目安は以下のとおりです。

- 一般的な植物 (野菜、花、観葉植物)
  - 発芽期：0.4～0.8 kPa (湿った環境が好ましい)
  - 成長期：0.8～1.2 kPa (蒸散と成長のバランスが良い)
  - 開花・結実期：1.0～1.5 kPa (やや乾燥気味で栄養吸収を促進)
- 熱帯植物 (例：ラン)：0.5～1.0 kPa (高湿度を好む)
- 乾燥に強い植物 (例：サボテン)：1.5～2.0 kPa でも耐える

### 不快指数とエアコンの目安

不快指数 (DI: Discomfort Index) は、気温と相対湿度を基に算出される指標で、人間が感じる暑さや不快感を数値化したものです。

- DI < 60: 寒いと感じる。エアコン (暖房) が必要な場合も。
- DI 60～65: 涼しく快適。エアコン不要。
- DI 65～70: 快適な範囲。ほとんどの人がエアコン不要。
- DI 70～75: やや暑く感じる。一部の人には不快。扇風機や通気で対応可能。
- DI 75～80: 暑さを感じる。エアコン (冷房) を検討。特に高齢者や暑さに弱い人は入れるべき。
- DI 80～85: かなり暑い。ほとんどの人がエアコンを入れた方が快適。
- DI 85 以上: 非常に不快で熱中症リスクが高まる。エアコンを強く推奨。

OLED は、左列に気温・WBGT・DI を大きい文字で、右列に測定気圧・推定標高・相対湿度・絶対湿度・VPD を小さい文字で表示します。
STA 接続中は最上段に設定ページへ接続する STA 側 IP アドレスを表示し、その下に日付・曜日・時刻を表示します。STA 未接続時は IP アドレスを表示しません。

### SoftAP と設定ページ

SoftAP `ESP32AP` は端末固有の12文字のアクセスキーを使う WPA2-PSK で保護されます。アクセスキーは小文字と数字で構成し、紛らわしい `l`、`o`、`0`、`1` は使用しません。起動後120秒間、OLED に SSID、アクセスキー、接続先 `192.168.4.1`、Web ユーザー名 `admin` を表示します。

設定ページは SoftAP 側と STA 側の両方からアクセスでき、全ページで HTTP Digest 認証を要求します。Web パスワードには SoftAP と同じアクセスキーを使用します。保存済み Wi-Fi パスワード、New Relic Ingest Key、Grafana API Key は設定ページへ再表示されず、空欄のまま保存すると現在値を維持します。設定を削除してもアクセスキーは維持されます。

STA 側の設定ページは平文 HTTP です。信頼できるローカルネットワーク内でのみ使用してください。

### 推定標高

気象庁の AMeDAS JSON から東京の当日12時の海面気圧を毎日13時以降に取得します。失敗時は前回値を維持し、1時間間隔で再試行します。

取得成功時の BME280 の測定気温・測定気圧と海面気圧を気圧高度式へ入力し、推定標高を1日1回算出して EEPROM に保存します。OLED、New Relic、Grafana Cloud は翌日の更新までこの値を使用します。固定基準気圧 `1013.25 hPa` を使う BME280 の高度 API は使用しません。東京と設置場所の気圧差を含むため、表示値は測量用途ではありません。

### New Relic へのメトリクス送信

設定ページでは、Wi-Fi 設定とともに次の New Relic 設定を保存できます。

- deviceId: 1〜32 文字の半角英数字
- location: `home`、`apartment`、`outdoor`、`office` から選択
- New Relic Ingest Key

Ingest Key は保存後に設定ページへ再表示されません。空欄のまま設定を保存した場合は、すでに保存されているキーを維持します。設定を削除すると Wi-Fi 設定、湿度補正値、New Relic 設定がすべて消去されます。

deviceId、location、Ingest Key が設定され、Wi-Fi 接続と NTP 時刻同期が完了している場合、5 分間隔で `https://metric-api.newrelic.com/metric/v1` へ次の gauge メトリクスを HTTPS POST します。

- `home.weather.temperature`: 気温 (°C)
- `home.weather.humidity`: 相対湿度 (%RH)
- `home.weather.absoluteHumidity`: 絶対湿度 (g/m³)
- `home.weather.pressure`: 気圧 (hPa)
- `home.weather.altitude`: 東京の当日12時の海面気圧から日次算出した推定標高 (m)
- `home.weather.vpd`: VPD (kPa)
- `home.weather.wbgt`: WBGT (°C)
- `home.weather.discomfortIndex`: DI

各メトリクスには共通属性として deviceId、location、`sensor=BME280` が付与されます。timestamp は NTP 同期後の Unix 秒です。いずれかの値が有効な有限値でない場合は送信しません。

### Grafana Cloud へのメトリクス送信

設定ページの Grafana API Key 欄からキーを保存できます。保存後は設定ページへ再表示されず、空欄のまま保存すると現在値を維持します。

deviceId、location、Grafana API Key が設定され、Wi-Fi 接続と NTP 時刻同期が完了している場合、New Relic と同じ8個の gauge 値を5分間隔で Grafana Cloud OTLP metrics endpoint へ HTTPS POST します。各データポイントには deviceId、location、`sensor=BME280` が付与されます。Grafana Cloud ではメトリクス名の `.` が `_` へ変換されるため、たとえば `home.weather.temperature` は `home_weather_temperature` として検索します。

設定を削除すると Grafana API Key も消去されます。API Key はソースコード、OLED、シリアル出力、HTTP 応答、OTLP payload には出力されません。

### WBGT と熱中症予防の目安

WBGT (暑さ指数) は、熱中症予防を目的とした温熱指標です。本機は気温 `T` (°C) と相対湿度 `RH` (%) から Stull の近似式で湿球温度 `Tw` (°C) を求め、屋内向けの式で WBGT を算出します。

```text
Tw = T × atan(0.151977 × sqrt(RH + 8.313659))
     + atan(T + RH) - atan(RH - 1.676331)
     + 0.00391838 × RH^(3/2) × atan(0.023101 × RH) - 4.686035
WBGT = 0.7 × Tw + 0.3 × T
```

OLED では、WBGT を左列の中央に表示します。直射日光や強い放射熱、気流は考慮しないため、屋外や放射熱のある場所では参考値として扱ってください。

| Room WBGT    | 判定     | 動作                                 |
| ------------ | -------- | ------------------------------------ |
| 25未満       | 経過観察 | 換気・扇風機。温度上昇を監視         |
| 25以上28未満 | 冷房推奨 | エアコンを入れるよう音声・表示で通知 |
| 28以上       | 冷房必須 | 本人の感覚にかかわらずエアコンを使用 |
| 31以上       | 危険     | 強い警告、家族等への通知を検討       |

## ハードウェア

- Arduino Nano ESP32
- Waveshare BME280 温湿度計センサー
- Waveshare SSD1306 OLED
- ブレッドボード

## 検証

標高式と気象庁セルの品質判定は、標準ライブラリだけの自己検証で確認できます。

```sh
python3 tests/test_altitude_and_jma.py
python3 tests/test_access_security.py
```

Nano ESP32 向けコンパイルは次のコマンドで実行します。

```sh
arduino-cli compile --fqbn esp32:esp32:nano_nora .
```
